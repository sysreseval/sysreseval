```
Key directories (production):
  /opt/sre/          — installation root
  /opt/sre/lab/      — lab definition files (srelab.py modules)
  /opt/sre/lib/      — shared libraries (ips.py, exetests.py, net_config.py, …)
  /var/lib/sre/      — runtime state
  /var/lib/sre/projects/  — one directory per running lab instance
  /var/lib/sre/archives/  — default evaluation archive destination
  /var/lib/sre/exam.json  — exam configuration (absent = no exam)
```

## Two-Tier Command Model

Every command that touches Docker must run as user `sre` (uid 1100). The `sre-wrapper` binary sets the `USER_USERNAME` environment variable and calls `sudo sre --user`, which then applies the action on behalf of the named user. The CLI binary itself drops from root to `sre_uid` before executing any logic.
